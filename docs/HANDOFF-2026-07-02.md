## What This Project Is

A Next.js 16 operational dashboard for managing an AI-powered development workflow. It provides:

- **Human-in-the-loop gates** — approve/reject/request-more-info on AI-generated tactics
- **PR lifecycle management** — sync from GitHub, CI status, merge with gate
- **Agent orchestration** — register AI agents, link skills, manage instructions
- **Strategic planning** — goals → projects → strategies → tactics hierarchy
- **Eisenhower Matrix** — importance/urgency task prioritization
- **Brain dump** — quick-capture ideas with LLM-powered triage
- **Decision log** — browseable history of all human decisions
- **Skills library** — centralized skill definitions linked to agents
- **System monitoring** — health checks for 7+ homelab services, activity timeline
- **External integrations** — Hermes ACP (agent framework), OpenCode (dev tooling), WireGuard watchdog

## Database Schema (Complete)

### Tables in `hil_ops`

| Table | Purpose | Key Columns |
|---|---|---|
| `users` | Operator identity (single-user homelab) | id, name, email |
| `agents` | AI agent registry | name, role, model, avatarUrl, **instructions**, **capabilities**, **status**, **icon**, **description** |
| `skills` | Skill definitions | name, description, content (markdown), tags (text[]) |
| `agent_skills` | Agent↔skill join table | agentId, skillId (composite PK) |
| `goals` | Strategic objectives | name, description, type, status, category, targetDate, parentGoalId, milestones |
| `projects` | Workstreams under goals | name, description, goalId, status, priority, color, tags |
| `strategies` | High-level strategies | title, goalId |
| `tactics` | Individual actions — **core table** | title, status, source, sourceId, confidence, riskLevel, **importance**, **urgency**, **collaborators**, goalId, estimatedMinutes, actualMinutes, acceptanceCriteria, requiresHumanApproval |
| `approvals` | Human decisions | tacticId, decision, notes, decidedBy |
| `messages` | Human↔agent threads | threadType, threadId, role, content |
| `brain_dump` | Quick-capture ideas | content, source, status, triageSummary, suggestedType |

### Enums

`goal_type`, `goal_status`, `goal_category`, `project_status`, `project_priority`, `brain_dump_source`, `brain_dump_status`, `brain_dump_convert_type`

## Design Conventions

- **Panel style:** `rounded-2xl border border-slate-200 bg-white dark:bg-slate-900 p-6 shadow-sm`
- **Color system:** slate (neutral), indigo (primary), emerald (success), amber (warning), rose (danger), sky (info), violet (accent)
- **Dark mode:** `@custom-variant dark (&:where(.dark, .dark *))` — class-based, toggled via localStorage
- **Server components:** All data-fetching panels are async RSCs. Client components only for interactivity.
- **Error handling:** `SectionErrorBoundary` wraps each panel. API routes return `{ ok, data?, error? }` JSON.
- **Icons:** `lucide-react` exclusively.
- **Timestamps:** `date-fns/formatDistanceToNow` for relative.
- **Type safety:** Exported types from shared components to avoid drift (e.g., `AgentBase` from `agent-editor.tsx`).
- **Optimistic updates:** Client components do optimistic UI + rollback on API failure.
